Introdução¶

Esse prottipo busca treinar, avaliar e testar modelos de classificao de textos em sentimentos (positivo e negativo).
Para realizar o treino e validao, foi utilizado um conjunto de dados abertos de avaliaes de produtos, o B2W-Reviews01, que contm mais de 130 mil avaliaes de clientes de comrcio eletrnico, coletadas do site Americanas.com entre janeiro e maio de 2018.

Pré processamento das avaliações¶

O pr-processamento de texto uma etapa crtica na classificao de texto e em tarefas de processamento de linguagem natural (NLP) em geral. Existem vrias razes pelas quais o pr-processamento de texto importante:

  • Normalizao de Dados: O pr-processamento ajuda a normalizar os dados textuais, o que significa que ele coloca os textos em um formato consistente e padronizado. Isso importante porque diferentes textos podem conter formatao, capitalizao e estruturas diferentes, o que pode dificultar a comparao e anlise.

  • Reduo de Rudo: Textos muitas vezes contm informaes irrelevantes, como espaos em branco extras, pontuao, caracteres especiais e tags HTML. O pr-processamento ajuda a eliminar esses elementos indesejados, o que melhora a qualidade dos dados e evita que o modelo se distraia com informaes desnecessrias.

  • Remoo de Stopwords: Stopwords so palavras comuns, como "e", "em", "o", "a", "de", "que", que no contribuem significativamente para o significado do texto. A remoo de stopwords reduz a dimensionalidade dos dados e ajuda o modelo a se concentrar em palavras mais informativas.

  • Stemming ou Lematizao: Stemming e lematizao so tcnicas para reduzir as palavras sua forma raiz. Isso ajuda a agrupar palavras semelhantes e a lidar com variaes morfolgicas. Por exemplo, "correr" e "correndo" podem ser reduzidos a "corr" pela tcnica de stemming.

  • Eliminao de Variaes de Palavras: O pr-processamento tambm ajuda a lidar com variaes de palavras, como singular/plural, tempos verbais e conjugaes. Isso garante que o modelo possa reconhecer palavras relacionadas como equivalentes.

  • Melhoria da Eficincia Computacional: Ao reduzir a quantidade de dados e melhorar a qualidade, o pr-processamento torna a anlise de texto mais eficiente em termos de recursos computacionais.

import re
from nltk.corpus import stopwords
from nltk.stem import SnowballStemmer

def preprocess_text(text):
    # Remoo de caracteres especiais e nmeros
    text = re.sub(r'[^a-zA-Z]', ' ', text)
    # Remoo de stopwords
    words = text.lower().split()
    words = [word for word in words if word not in stopwords.words('portuguese')]
    # Stemming
    stemmer = SnowballStemmer('portuguese')
    words = [stemmer.stem(word) for word in words]
    return ' '.join(words)

b2wReviews1['processed_text'] = b2wReviews1['review_text'].apply(preprocess_text)

Vetorização¶

A vetorizao de textos um processo fundamental na anlise de sentimentos, que envolve a transformao de texto em dados numricos que podem ser processados por algoritmos de aprendizado de mquina. Isso permite que os modelos de anlise de sentimentos compreendam e classifiquem o contedo de texto com base em emoes, opinies e polaridade, como positivo, negativo ou neutro.

Para esse trabalho, vamos utilizar a tcnica TF-IDF, que atribui um valor a cada palavra com base em sua frequncia no documento e na frequncia inversa em todo o corpus de documentos. Isso ajuda a destacar palavras importantes e reduzir o peso de palavras comuns.

Treinando e avaliando modelos¶

Com os dados devidamento preparados, seguimos para a gerao e avaliao dos modelos. Para isso, vamos gerar uma lista de modelos utilizando diferentes algoritmos de classifcao, e para cada um vamos realizar algumas mtricas no conjunto de validao. As mtricas utilizadas sero:

  • Accuracy (Acurcia): A acurcia uma mtrica simples que mede a proporo de previses corretas em relao ao nmero total de previses. til para avaliar o desempenho geral de um modelo de classificao, especialmente quando as classes tm tamanhos aproximadamente iguais. No entanto, a acurcia pode ser enganosa em casos de desequilbrio de classes, onde uma classe muito mais predominante que a outra.

  • F1 Micro (F1 Micro-averaged): O F1 Micro uma medida de desempenho que calcula a mdia harmnica das mtricas de preciso e recall a nvel de instncia e adequado para problemas de classificao multiclasse. Ele leva em considerao as previses corretas e incorretas para todas as classes, dando igual peso a cada classe. til quando o desequilbrio de classes um problema.

  • F1 Macro (F1 Macro-averaged): O F1 Macro semelhante ao F1 Micro, mas calcula a mdia harmnica das mtricas de preciso e recall para cada classe separadamente e, em seguida, faz uma mdia simples desses valores. Diferentemente do F1 Micro, o F1 Macro d igual peso a cada classe, independentemente do tamanho da classe. Isso o torna adequado quando voc deseja avaliar o desempenho de um modelo em todas as classes de forma equitativa.

  • Precision (Preciso): A preciso mede a proporo de instncias classificadas como positivas que eram verdadeiramente positivas. Em outras palavras, a capacidade do modelo de no fazer falsos positivos. Uma alta preciso significa que o modelo bom em classificar apenas as instncias que so realmente positivas.

  • Recall (Revocao): A revocao mede a proporo de instncias positivas que foram corretamente classificadas como positivas. a capacidade do modelo de identificar todas as instncias positivas, minimizando os falsos negativos. Uma alta revocao importante em cenrios onde identificar todas as instncias positivas crtico.

from sklearn.linear_model import LogisticRegression
from sklearn.tree import DecisionTreeClassifier
from sklearn.ensemble import RandomForestClassifier
from sklearn.naive_bayes import MultinomialNB
from sklearn.model_selection import KFold, cross_validate
from sklearn.metrics import accuracy_score, precision_score, f1_score, recall_score

models = []
models.append(('MultinomialNB', MultinomialNB()))
models.append(('LogisticRegression', LogisticRegression(random_state=1, tol=1e-5,max_iter = 100000)))
models.append(('DecisionTreeClassifier', DecisionTreeClassifier()))
models.append(('RandomForestClassifier',RandomForestClassifier()))

metrics = ['accuracy', 'f1_micro', 'f1_macro', 'precision', 'recall']
results_df = pd.DataFrame(columns=['model'] + metrics)

for name, model in models:
    model.fit(X_train_vect, y_train)
    y_pred = model.predict(X_val_vect)
    accuracy = accuracy_score(y_val, y_pred)
    f1_micro = f1_score(y_val, y_pred, average='micro')
    f1_macro = f1_score(y_val, y_pred, average='macro')
    precision = precision_score(y_val, y_pred, average='weighted')
    recall = recall_score(y_val, y_pred, average='weighted')
    results_df = pd.concat([results_df, pd.DataFrame({'model': [name], 'accuracy': [accuracy], 'f1_micro': [f1_micro], 'f1_macro': [f1_macro], 'precision': [precision], 'recall': [recall]})], ignore_index=True)
